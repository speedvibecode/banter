# BANTER — UI / UX Design Document (MVP)

## Purpose

Define the UI and interaction principles for the Banter MVP so the product feels:

* fast
* decisive
* addictive
* argument-centric

The UI must make arguments and verdicts the **center of the experience**.

Users should feel they are participating in a **crowd judgment engine**, not just a poll app.

---

# 1. Core UI Philosophy

The interface should communicate three ideas immediately:

1. This is where arguments get settled
2. Voting expresses conviction
3. Every poll ends with a verdict

The UI must feel closer to:

Twitter debate × prediction market × Reddit discussion

Not like:

SurveyMonkey
Google Forms

---

# 2. Visual Identity

## Tone

Playful but authoritative.

The platform is a **judge**, but also part of internet culture.

Avoid corporate design.

Prefer bold typography and clear verdict signals.

---

## Core Colors

Primary:

Black / Dark gray background

Accent:

Electric blue

Secondary accent:

Orange (for conflict)

Success:

Green (winning side)

Loss:

Red (losing side)

---

# 3. Typography

Headers:

Bold
Large
High contrast

Poll titles must be extremely readable.

Example sizes:

Poll title: 24–28px
Options: 18–20px
Metadata: 14px

---

# 4. Core Screens

MVP requires 7 screens.

1. Landing page
2. Poll feed
3. Poll page
4. Voting interface
5. Result page
6. Create poll page
7. Profile page

---

# 5. Landing Page

Goal: explain the product instantly.

Layout:

Hero text

"Where arguments end."

Subtext

"Post the debate. Let the crowd decide."

Primary CTA:

Create Poll

Secondary CTA:

View Arguments

Below hero show:

Example polls

---

# 6. Poll Feed

This is the **core engagement screen**.

Each poll displayed as a card.

Poll Card layout:

Title
Option A
Option B
Vote totals
Time remaining

Visual bar showing support split.

Example:

Option A 63% | Option B 37%

Users must be able to open a poll with one click.

---

# 7. Poll Page

This page hosts the argument.

Top section:

Poll title
Poll description
Creator

Middle section:

Voting interface

Bottom section:

Vote distribution
Comments (optional MVP)

---

# 8. Voting Interface

This is the **most important UI component**.

Users must feel they are expressing conviction.

Design:

Two sliders or allocation bars.

Total points available: 100

Example layout:

Option A
[slider]

Option B
[slider]

Remaining points indicator.

Example:

Points remaining: 20

---

# 9. Poll Status Indicator

Every poll shows status:

Active
Closing soon
Closed

Closing soon = last 10% of time.

---

# 10. Result Page

When poll closes, show a verdict.

Large verdict banner:

"Banter has spoken"

Winner highlighted.

Example:

YES — 62%
NO — 38%

Below:

Total votes
Total points

Share button prominently visible.

---

# 11. Create Poll Page

Creation must be extremely fast.

Fields:

Title
Option A
Option B
Duration
Category

UI must feel like composing a tweet.

---

# 12. Profile Page

User info displayed:

Username
Reputation
Polls participated
Polls created

Optional section:

Prediction accuracy

---

# 13. Micro-Interactions

Important for addictiveness.

Examples:

Vote submitted animation

Result reveal animation

Close poll countdown

Reputation gain animation

Example message:

"+42 reputation"

---

# 14. Mobile Design

Most usage will be mobile.

Key rules:

Poll cards must fit mobile screens

Voting sliders must be thumb friendly

Share button easy to access

---

# 15. Result Card (Share Image)

This is the **viral engine**.

Design:

Poll title
Result
Percentages
Vote count

Brand line:

"Banter has spoken"

Must look good in screenshots.

---

# 16. Design Priorities

Priority order:

1. Voting UI clarity
2. Poll feed readability
3. Result card shareability
4. Poll creation speed

Everything else is secondary.

---

# 17. MVP Simplicity Rules

Avoid:

Complex animations
Heavy graphics
Too many colors

Speed and clarity matter more.

---

# 18. Final UX Principle

Users should feel like they are:

casting a judgment

not filling a form.

The interface should constantly reinforce the idea:

"The crowd is deciding."
